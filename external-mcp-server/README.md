# Standalone External Medical MCP Server

This is a production-quality, completely independent Model Context Protocol (MCP) server that mimics a public healthcare resource information provider. It exposes directories of clinical inventory (blood bank units, organ donor matching, medicines catalog, drug interactions, surgical equipment, hospital listings, suppliers, and transport logistics) via standard MCP tools.

It operates strictly as a **data provider** with clean separation of concerns, loading data dynamically from a normalized SQLite database. It contains **no scheduling, planning, or clinical coordination logic**.

---

## 🏛️ Architecture & Project Structure

The project directory is structured as follows:

```text
external-mcp-server/
│
├── api/
│   └── index.py            # FastAPI ASGI entrypoint, mounts MCP http app
│
├── database/
│   ├── db_setup.py         # Table schemas definition and mock data seeding
│   └── medical_resources.db# Live SQLite database file (generated by db_setup)
│
├── models/
│   └── schemas.py          # Pydantic schemas validating SQLite row data
│
├── repository/
│   └── database_repo.py    # Database connection manager and SQL query abstraction
│
├── tools/
│   └── mcp_tools.py        # FastMCP instance and tools registrations
│
├── requirements.txt        # Server dependencies list
├── vercel.json             # Vercel Serverless routing rewrites
└── README.md               # Setup and testing guide
```

## 📋 Tool Reference Index

The following MCP tools are exposed to AI clients:

| Tool Name | Parameters | Description |
| :--- | :--- | :--- |
| `search_blood_inventory` | `blood_type` (str), `hospital_id` (str, optional) | Query the clinical blood bank database. |
| `search_organ_registry` | `organ_type` (str), `hospital_id` (str, optional) | Check organ donor lists and tissue matching. |
| `search_medicine` | `query` (str) | Search the generic/brand drug catalog. |
| `check_drug_interactions`| `medications` (List[str]) | Run a pairwise scan for contraindications. |
| `search_equipment` | `name` (str, optional), `hospital_id` (str, optional) | Fetch surgical equipment and sterilization states. |
| `search_hospital` | `query` (str) | Look up coordinates, contact info, and addresses. |
| `search_supplier` | `query` (str) | List medical logistics suppliers and lead times. |
| `get_storage_requirements`| `resource_type` (str), `item_id` (str) | Retrieve temperatures and handling requirements. |
| `check_resource_availability`| `resource_type` (str), `identifier` (str) | General status check for specific serial IDs. |
| `get_transport_information`| `source_hospital_id` (str), `destination_hospital_id` (str) | Look up vehicle types (ambulance, helicopter) and ETAs. |

## 🚀 How to Run Locally

### 1. Pre-requisites & Setup
Ensure Python 3.11+ is installed.

```bash
# Navigate to the server folder
cd external-mcp-server

# Create virtual environment (if not already done)
python3 -m venv .venv

# Activate virtual environment
source .venv/bin/activate

# Install package dependencies
pip install -r requirements.txt
```

### 2. Generate and Seed the Database
Before running the server, build the SQLite schema and seed mock data:
```bash
python3 database/db_setup.py
```
This generates the file `database/medical_resources.db`.

### 3. Start the Server
Launch the FastAPI development server:
```bash
# Ensure the virtual environment is active:
source .venv/bin/activate

# Run the server using the virtual environment's python/uvicorn
python3 -m uvicorn api.index:app --reload --port 8000
```
The server will start on `http://localhost:8000`. You can visit `http://localhost:8000/health` to verify the service status.

### 4. Interactive Testing with MCP Inspector
To inspect and call your tools interactively:
```bash
npx @modelcontextprotocol/inspector http://localhost:8000/mcp/sse
```
*Note: Since FastMCP mounting is used, the transport path is set to `/mcp/sse` and `/mcp/messages`.*

## ⚡ Deployment to Vercel

This repository is ready to deploy as a Vercel Serverless Function:

1.  **Configure Git**: Ensure the `external-mcp-server` folder is tracked and pushed to your git repository.
2.  **Import on Vercel**:
    *   Log in to the [Vercel Dashboard](https://vercel.com).
    *   Click **Add New Project** and select your repository.
    *   In the project configuration page, set the **Root Directory** to `external-mcp-server`.
    *   Keep the default Build and Development settings (Vercel automatically detects requirements.txt and sets up the serverless function handler at `api/index.py`).
3.  **Deploy**: Click **Deploy**.
4.  **Connect**: Your remote production MCP server will be live. Connect using:
    ```bash
    npx @modelcontextprotocol/inspector https://your-project.vercel.app/mcp/sse
    ```
